#  Movie Search App (Async/Await + API)
##  Опис проєкту
Movie Search App — це веб-додаток для пошуку фільмів з використанням **OMDB API**.
Користувач може знаходити фільми за назвою, переглядати детальну інформацію, фільтрувати результати та зберігати обрані фільми у список (Watchlist).
Проєкт реалізований з використанням **async/await**, Fetch API та сучасних підходів до роботи з асинхронним кодом.

##  Функціональні можливості
###  Пошук фільмів
* Пошук за назвою
* Debounce (затримка 500мс для оптимізації запитів)

###  Інформація про фільм
* Постер
* Назва
* Рік
* Рейтинг IMDb

### Фільтрація
* За роком (через API)
* За рейтингом (на клієнті)

### ♾ Infinite Scroll
* Автоматичне підвантаження нових фільмів при прокрутці

###  Watchlist
* Збереження обраних фільмів у localStorage
* Автоматичне відновлення списку

##  Технічні особливості
* ✅ Async/Await (без .then())
* ✅ Fetch API для HTTP запитів
* ✅ Try/Catch для обробки помилок
* ✅ Debounce для оптимізації
* ✅ Infinite Scroll
* ✅ LocalStorage (Watchlist)
* ✅ Loading state (індикатор завантаження)
* ✅ Error handling (повідомлення про помилки API)

##  Використаний API
**OMDB API**
* Реєстрація: http://www.omdbapi.com/apikey.aspx
* Пошук:
```
http://www.omdbapi.com/?apikey={key}&s={title}
```

##  Як запустити проєкт
1. Завантажити або клонувати репозиторій:
```
git clone https://github.com/your-username/js-async-api-variant3
```
2. Отримати API ключ:
* Зареєструватися на OMDB
* Активувати ключ через email
3. Вставити ключ у файл `app.js`:
```js
const API_KEY = "your_api_key";
```
4. Відкрити файл:
```
index.html
```

##  Як користуватись
1. Ввести назву фільму (наприклад: `batman`)
2. Переглянути результати
3. Використати фільтри:
   * Year
   * Min Rating
4. Прокрутити вниз для завантаження нових результатів
5. Додати фільми у Watchlist

##  Відомі обмеження
* OMDB API має ліміт запитів (~1000/день)
* Рейтинг доступний тільки через окремий запит (details)
* Жанри не підтримуються у пошуку API

## Оптимізації
* Використано debounce для зменшення кількості запитів
* Деталі фільмів завантажуються окремо
* Можливе кешування для зменшення навантаження на API

##  Що реалізовано 
У проєкті реалізовано:
* Асинхронні запити через async/await
* Робота з зовнішнім API (OMDB)
* Обробка помилок (try/catch)
* Оптимізація запитів (debounce)
* Динамічне оновлення UI
* Робота з localStorage
